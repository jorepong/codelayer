# 산출물 예시 — 알림 플랫폼 "단계 7: 메일 발송"

실제 Spring 프로젝트(공고 수집 → 매칭 → 알림 생성 → **메일 발송**)의 한 단계를 CodeLayer로 학습하며 나온 **진짜 산출물**입니다. AI가 지어낸 예시가 아니라, `.codelayer/`가 실제로 만들어 낸 파일을 그대로 옮겨 왔습니다.

> 이 폴더는 **결과물을 보여주기 위한 쇼케이스**일 뿐 템플릿의 일부가 아닙니다. 새 프로젝트에 얹을 때 복사하는 것은 `.codelayer/`·`.plan/`·`CLAUDE.md`이며, `examples/`는 가져가지 않습니다.

한 단계가 남기는 산출물 전체가 여기 다 있습니다. **개요에서 시작해 아래 순서로 보면** CodeLayer가 어떻게 도는지 한 바퀴가 그려집니다.

| 파일 | 산출물 종류 | 무엇인가 |
|------|------------|----------|
| [`overview.md`](overview.md) | **통합 개요** | 단계 완료 시 생성. 이 단계의 학습거리 전체 — 개념 학습 포인트(`[N]`·추천 점수)와 구현 유닛(`[U N]`·장악 점수)를 한 표에 펼치고, 구현 여정을 요약한다. **학습의 출발점.** |
| [`journal.md`](journal.md) | **구현 일지** | 구현 *중에* 남긴 날것의 기록(손댄 순서·버린 대안·확인한 곳·만난 오류). 학습자용이 아니라, 아래 구현 관전 문서를 쓰기 위한 원재료. |
| [`learn/테스트더블MockitoBean/`](learn/테스트더블MockitoBean/) | **개념 학습** (`[2]`) | 개요에서 개념을 고르면 생성. 한 개념을 섹션별 문서로 깊게 설명한다(튜터 목소리·비유·코드 앵커 `❶❷❸`·이해도 평가 `exercises/`). `assets/`에 직접 그린 SVG 시각 자료. |
| [`learn/구현여정/`](learn/구현여정/) | **구현 장악** (`[U1]`) | 개요에서 구현 유닛을 고르면 생성. 완성 코드 설명이 아니라 그 코드가 *만들어지는 과정*을 시간 순서로 관전한다(예측 게이트 🔮·버린 대안·멈춤/빨리감기). `exercises/`에 결함 주입·재구성 과제. |

## 이렇게 보면 좋아요

1. **[`overview.md`](overview.md)** 의 맨 위 요약 표를 먼저 봅니다 — 이 단계에서 뭘 배울 수 있고, 각 항목이 지금 학습자에게 얼마나 값진지(점수·배지)가 한눈에 들어옵니다.
2. 개념이 궁금하면 **[`learn/테스트더블MockitoBean/01-왜-가짜가-필요한가.md`](learn/테스트더블MockitoBean/01-왜-가짜가-필요한가.md)** — "왜 필요한가"에서 출발하는 개념 설명이 어떤 결인지 보입니다.
3. 구현 흐름을 직접 장악하는 트랙이 궁금하면 **[`learn/구현여정/01-커밋된-것만-보낸다.md`](learn/구현여정/01-커밋된-것만-보낸다.md)** — 코드가 짜이는 과정을 옆에서 지켜보는 "관전" 문서입니다. 옆에 [`journal.md`](journal.md)를 같이 두면, 날것의 일지가 어떻게 관전 문서로 자라는지 대비됩니다.
4. 배운 걸 능력으로 바꾸는 자리는 **[`learn/구현여정/exercises/Ex01-발송흐름조립-결함주입.md`](learn/구현여정/exercises/Ex01-발송흐름조립-결함주입.md)** — "컴파일도 되고 정상 흐름 테스트도 통과하는데 결함이 하나 숨은" 코드를 잡아내는 과제입니다.
